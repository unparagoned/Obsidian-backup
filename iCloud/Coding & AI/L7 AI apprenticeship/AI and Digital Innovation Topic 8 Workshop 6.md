### Prompting

Chain of thought

Can tell it to give outputs in a structured way, e.g. I want these columns

Zero shot means no example

1. Give Context i.e. explain role

2. Explain expectation i.e. return this in this format/find this in this
3. Explain Edge cases i.e. dont return null

Ask it to beat or improve on existing function, etc. Could use different models.

Direct injection – User input includes new instructions. Instruction override – Tells model to ignore prior prompts.

Obfuscated input – Uses hidden characters or formatting.

Encoded payload – Malicious input in base64 or hex.

Data poisoning – Malicious data added to training or context.

Prompt smuggling – Hides instructions in file metadata or tags. Context overflow – Overloads context to push out original prompt. Prompt chaining abuse –

Misuses intermediate steps to inject. Masking as questions –

Malicious prompt disguised as a query.

System prompt leakage – Uses tricks to reveal or alter system prompt.

Jailbreaking – Bypasses filters using clever phrasing.

 Nested prompts – Injection hidden in embedded sub-prompts.
