# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repository Overview

This is an **Obsidian knowledge vault** containing study materials for a Level 7 AI Data Specialist Apprenticeship (UK IfATE standard st0763), delivered via QA Learning. It contains ~96 markdown files covering learning notes, assignments, workshop exercises, and EPA (End-Point Assessment) preparation. There is no build, test, or lint infrastructure.

## Repository Structure

Files are organized by module:

- **AI and Data Science Professional Practice** — Professional competency, ethics, workplace reflection
- **Data Science Principles** — Statistics, data architecture, governance, ML fundamentals
- **Programming for AI** — Python-based ML: sklearn, feature engineering, time series, visualization
- **Machine Learning using Cloud Computing** — Deep learning (CNNs, RNNs, LSTMs), AWS (S3, Athena, SageMaker)
- **AI and Digital Innovation** — Transformers, BERT, LLMs, NLP applications
- **Disruptive Leadership for Sustainable Strategy** — DataOps/MLOps, change management, data strategy

**Z-prefixed files** are priority reference materials:
- `Z EPA notes.md` — EPA structure, assessment criteria, portfolio/journal guidance
- `Z Technical test.md` — Practice technical test (scenario-based, 100-min format)
- `Z Technical test BCS L7...` — Official BCS sample paper
- `Z project.md` — Project planning (TFIDF, SVM, NN, BERT approaches)
- `Z Assignment - planning and writing.md` — Assessment writing strategies

## Git Workflow

Commits are automated vault backups. The main branch for PRs is `newbranch`.

## Key Context

- **Assessment format:** Technical test (4 long-response questions, 100 min), professional discussion (60 min), learner journal/portfolio, project report
- **KSB mapping:** Knowledge, Skills, and Behaviours are tracked across modules against the apprenticeship standard
- **Primary tools covered:** Python, pandas, numpy, scikit-learn, TensorFlow/Keras, SQL, AWS, BERT/transformers, CRISP-DM methodology
